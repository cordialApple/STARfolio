# Stage 12 — SOLID refactor sweep

Part of the [build plan](../build-plan.md) · Context to load: [ai-layer](../architecture/ai-layer.md) · [process-and-ipc](../architecture/process-and-ipc.md) · [voice](../architecture/voice.md) · [data-model](../architecture/data-model.md) · [privacy-and-risks](../architecture/privacy-and-risks.md)

Goal: pay down SOLID debt across main + renderer. Each step carves a pure seam out of a mixed-concern module (SRP), inverts a hardwired dep behind an optional injected collaborator (DIP — same pattern as `client?: ParseClient`), or dedups a duplicated harness (DRY/OCP). Every step is **one behavior-preserving PR**. No feature change, no IPC-surface change, no model-routing change. Payoff is testability: pure logic that today drags `better-sqlite3`/Electron/native binds into vitest becomes node-testable in its own folder.

Why its own stage: this is a wide sweep grounded in a two-pass ultracode scout (34 confirmed findings → draft → adversarial critique → final). Ordering follows testability leverage and blast radius: cheap string/relocation/type-only seams first to warm the pattern, AI-subsystem seams next, widest + most behavior-sensitive changes last. Two critique-driven splits shape it — the clean high-value interview→parse dedup (12.6) is separated from the questionable-altitude extract fold (12.19) so the good change isn't blocked on the risky one; and every string-converging shortcut is replaced by explicit copy preservation so "behavior-preserving" holds literally, not just "test-safe".

**Standing rails (every PR):** `getSecret('anthropic_api_key')` stays the only key path. IPC handlers never trust renderer-supplied fs paths (resolve from own DB); `saveDialog`/`openDialog` paths are dialog-derived, not renderer-supplied. Local-whisper-only (no Web Speech). Pure-logic tests import NO `better-sqlite3`/Electron/native bind — own-folder-green is the bar. Preserve all user-visible copy by assertion, not just by absence of a test.
